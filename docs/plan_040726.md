# SDLC/DPLL Support for SCC — AppleTalk over LocalTalk

## Status: DEFERRED — revisit after boot is working

## Context

The plan_040526.md fidelity pass is complete through Step 5c. Step 5d (SDLC + DPLL) is the final piece needed to enable AppleTalk over LocalTalk. The infrastructure is ready: `rtxc_en` (3.672 MHz) is wired into the SCC, WR registers are stored, and the async serial path is stable. This plan covers the full SDLC implementation.

LocalTalk is 230.4 kbit/s SDLC with FM-0 encoding. RTxC (3.672 MHz) = exactly 16x bit rate. Frames use 0x7E flags, zero-bit stuffing, CRC-16 CCITT, and SDLC addressing.

## Commit Plan

### Commit 1: Register Infrastructure (WR6, WR11, SDLC mode wire)

**No behavioral change.** Add write logic for WR6 (sync address) and WR11 (clock source select) in `rtl/scc.v`. Add `sdlc_mode_a/b` detection wires (`wr4_a[5:4] == 2'b10`). These registers are stored but not consumed yet.

- `scc.v`: Add `reg [7:0] wr11_a, wr11_b;` declarations. Add `always` blocks for WR6 and WR11 writes following existing WR5 pattern. Add `sdlc_mode_a/b` wires.
- Verification: boot smoke test passes unchanged.

### Commit 2: CRC-16 CCITT Module

**New file `rtl/sdlc/scc_crc16.v`.** Standalone serial LFSR: `clk`, `rst`, `enable`, `bit_in` -> `crc_out[15:0]`, `crc_valid`. Init 0xFFFF, polynomial 0x1021. Same module used by both TX (generate) and RX (validate -- residual 0x1D0F = good).

- Verification: known test vector "123456789" -> CRC 0x29B1 (Verilator `$display` check).

### Commit 3: FM-0 Encoder + Decoder

**New files `rtl/sdlc/scc_fm0_enc.v` and `scc_fm0_dec.v`.**

- **Encoder**: 4-bit phase counter (0..15) divides `rtxc_en` into bit periods. Always transition at boundary (phase 0). Data '0' -> also transition at phase 8. Data '1' -> no mid-point transition.
- **Decoder**: Edge detector on serial input. Classify transitions as boundary vs mid-point based on timing relative to recovered clock. Output data bits with valid strobe. Expose `transition_detect` for DPLL.
- Verification: loopback (encoder output -> decoder input), verify round-trip on known patterns.

### Commit 4: DPLL

**New file `rtl/sdlc/scc_dpll.v`.** 4-bit phase accumulator at `rtxc_en` rate. Resync +/-1/+/-2 on each FM-0 transition. Outputs `dpll_bit_clk_en` at optimal sample point. Clock-missing flags after 2+ bit periods with no transition.

- WR14[7:5] DPLL commands (enter search, reset missing clock, disable, set source) decoded in `scc.v` and passed as control inputs.
- Verification: FM-0 stream with slight jitter -> stable lock within 2-3 flag bytes.

### Commit 5: SDLC TX Path

**New file `rtl/sdlc/scc_sdlc_tx.v`.** State machine: IDLE -> FLAG -> DATA -> CRC1 -> CRC2 -> CLOSING_FLAG -> IDLE.

- Zero-bit insertion after 5 consecutive 1s (bypassed during flag transmission).
- CRC-16 computed over address + data, appended inverted (SDLC convention).
- TX underrun detection when CPU doesn't provide next byte in time -> EOM signal to scc.v.
- Instantiates `scc_fm0_enc` and `scc_crc16` internally.
- Verification: capture serial output, decode externally, verify valid SDLC frame.

### Commit 6: SDLC RX Path

**New file `rtl/sdlc/scc_sdlc_rx.v`.** State machine: HUNT -> SYNC -> RECEIVE -> CHECK.

- HUNT: search for flag (0x7E) in recovered bit stream, set Sync/Hunt status.
- SYNC: flag found, check address byte vs WR6 (or 0xFF broadcast). Address match -> RECEIVE.
- RECEIVE: deserialize, zero-delete, feed CRC, present bytes via `rx_data`/`rx_valid`.
- On closing flag: `frame_end`, check CRC residual (0x1D0F = good).
- Abort detection: 7+ consecutive 1s.
- Instantiates `scc_dpll`, `scc_fm0_dec`, `scc_crc16`.
- Verification: generate SDLC frame externally -> feed to RX -> verify byte output and CRC status.

### Commit 7: SCC Integration (Mode Muxing + RR Updates)

**The integration commit.** All gated behind `` `ifdef SCC_SDLC ``.

In `rtl/scc.v`:
1. **Instantiate** `scc_sdlc_tx` and `scc_sdlc_rx` for both channels.
2. **TX output mux**: `wire tx_out_a = sdlc_mode_a ? sdlc_tx_serial_a : uart_tx_internal_a;` (rename current `tx_internal_a` to `uart_tx_internal_a`).
3. **RX data mux**: `effective_rx_wr_a = sdlc_mode_a ? sdlc_rx_valid_a : rx_wr_a;` feeds existing RX FIFO.
4. **TX data routing**: WR8 write feeds `sdlc_tx_a` when in SDLC mode.
5. **RR0 updates**: bits 4 (Sync/Hunt), 6 (TX Underrun/EOM), 7 (Abort) from SDLC RX/TX.
6. **RR1 updates**: bits 5 (Overrun), 6 (CRC Error), 7 (End of Frame) from SDLC RX.
7. **RR10 updates**: bits 7:6 (clock missing) from DPLL.
8. **Remove** `_unused_rtxc_en` wire, pass `rtxc_en` to SDLC modules.

- Single-driver rule: all muxing via `wire` assignments, never two `always` blocks on same `reg`.
- Verification: boot smoke test passes (WR4[5:4] defaults to 00 = async, SDLC modules idle).

## Key Design Decisions

- **Separate modules** (not inline in scc.v): scc.v is already 1733 lines. SDLC datapath is independently testable.
- **All synchronous**: everything runs on `clk` (32.5 MHz) with `rtxc_en` as clock enable. No separate clock domain.
- **`ifdef SCC_SDLC` gating**: entire SDLC path excluded when undefined. Async path untouched.
- **Existing RX FIFO reused**: SDLC RX presents bytes through same enqueue interface as async rxuart.
- **WR11 decoded but initially hardwired**: TX from RTxC, RX from DPLL (the natural LocalTalk config). Full WR11 flexibility can come later.
