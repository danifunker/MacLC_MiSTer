# MacLC Verilator Simulation Architecture

This document describes the verilator simulation setup for the MacLC core.

## Overview

The verilator simulation allows running the MacLC FPGA core in software for debugging and development. It replaces MiSTer-specific hardware interfaces with simulation-friendly equivalents.

## File Structure

### Core Simulation Files

| File | Description |
|------|-------------|
| `sim.v` | Top-level simulation wrapper adapted from MacLC.sv |
| `sim_ram.v` | Simple 8MB synchronous RAM replacing SDRAM |
| `sim_main.cpp` | C++ testbench with GUI (ImGui) |
| `Makefile` | Build configuration for verilator |

### Support Libraries (in `sim/`)

| File | Description |
|------|-------------|
| `sim_bus.cpp/h` | ROM/disk loading via ioctl interface |
| `sim_blkdevice.cpp/h` | SCSI block device emulation |
| `sim_video.cpp/h` | Video output to SDL window |
| `sim_audio.cpp/h` | Audio output |
| `sim_input.cpp/h` | Keyboard/mouse input handling |
| `sim_clock.cpp/h` | Clock generation |
| `sim_console.cpp/h` | Debug console |

## Architecture

### Clock Domain

- System clock: 32MHz (provided by testbench)
- 8MHz enable: Generated by dividing clk_sys by 4
- 16MHz enable: Generated by dividing clk_sys by 2

### Memory Map

The `sim_ram` module provides 8MB of unified memory:
- ROM area: Offset 0x100000 (word address)
- RAM area: Offset 0x000000
- Disk images: Offset 0x080000 and 0x100000

### Hardcoded Configuration

For simulation, the following settings are fixed:
- Machine mode: LC (not Plus)
- CPU: 68000 (FX68K)
- RAM: 4MB
- Video mode: 4bpp
- Monitor: 13" RGB (512x384)
- Turbo: Disabled (8MHz)

### Removed MiSTer Components

The following MiSTer-specific modules are not used in simulation:
- `hps_io` - Replaced by sim_bus for file loading
- `pll` - Clock enables generated directly
- `sdram` - Replaced by sim_ram
- `video_freak` - Not needed for simulation
- DDRAM interface - Not used
- SD card interface - Simulated by sim_blkdevice

## Building

```bash
cd verilator
make clean
make
```

Requirements:
- Verilator 4.x or 5.x
- SDL2 development libraries
- OpenGL development libraries
- C++17 compatible compiler

## Running

```bash
./obj_dir/Vemu
```

### Loading ROM

1. Click "Load ROM" button in the simulation control window
2. Select a Mac LC ROM file (.rom or .bin)
3. The ROM will be loaded via the ioctl interface

### Controls

| Key | Function |
|-----|----------|
| Arrow keys | Mouse movement |
| A | Left mouse button |
| B | Right mouse button |
| M | Menu |

## Data Widths

The simulation uses 16-bit data paths to match the MacLC hardware:

| Signal | Width | Description |
|--------|-------|-------------|
| ioctl_dout | 16-bit | ROM loading data |
| sd_buff_dout | 16-bit | SCSI read data |
| sd_buff_din | 16-bit | SCSI write data |
| sd_buff_addr | 8-bit | SCSI buffer word address |

## RTL Files Included

From `files.qip`:
- CPU: fx68k/*.sv, tg68k/*.v
- Memory: addrController_top.v, addrDecoder.v, dataController_top.sv
- Video: maclc_v8_video.sv, ariel_ramdac.sv, videoTimer.v, videoShifter.v
- I/O: via6522.sv, pseudovia.sv, adb.sv, ps2_kbd.sv, ps2_mouse.v
- Storage: iwm.v, floppy.v, floppy_track_encoder.v, scsi.v, ncr5380.sv
- Serial: scc.v, uart/txuart.v, uart/rxuart.v
- Other: rtc.v

## Debugging

### Waveform Tracing

The Makefile includes `--trace-fst` for FST waveform generation. Traces are saved to `traces/` directory.

### Memory Editor

The GUI includes a RAM editor window (currently shows placeholder - direct memory access requires verilator public signals).

### Debug Console

The debug log window shows:
- ROM loading progress
- Disk mount operations
- Simulation status
